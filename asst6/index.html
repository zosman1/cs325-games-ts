The Legend of Stone

  • Originality: What is original about your game? The player is able to explore a relatively large map, while being incentivised to score points, the player is completely free to do whatever they wish.
  • Technical Merit: What is interesting about your game technology? I wrote the entire game by hand, no examples, no stock images, no stock music. The game also has multiple high object physics groups. Because of this I was able to have the player collide with each individual object on the map. It was also nontrivial to have some of these objects (the apples) interact with the player in a different way than just normal collision, i was able to implement a callback that allowed a function to be called in the local scope when any apple interacts withe the player, that allows a special function handler to be run that allows me to increment the score and remove that specific apple that the player colided with. Additionaly, I was able to implement this rather big map without resorting to tiling the map, when reading the documentation, I found a camera zoom paramater and was able to use this to simulate the same experirence. The next problem that arose from this was having a score that stayed in the players view, I got stuck on this for a while but was able to have a dynamic text object that moves alongside the player.
  • Prototype Postmortem: What did you learn from this prototype? What was the easiest or hardest part of making it? The hardest part of this prototype was figuring out how I could get the player to move around the map without tiling. This took a lot of searching through the phaser documenation and eventually I discovered a camera zoom and a camera follow method that allowed me to simulate the behavior I wanted in the game view.
